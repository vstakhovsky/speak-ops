# Speaking Style Guide

**Purpose:** Define natural spoken Business English for SpeakOps.

## Core Principle

> **Optimize for natural spoken Business English, not formal written English.**

## What is Natural Spoken English?

Natural spoken English is:
- **Conversational** — How people actually speak in meetings
- **Direct but polite** — Clear without being rude
- **Rhythmic** — Natural flow and pauses
- **Context-appropriate** — Fits IT/Product/AI workplace
- **Authentic** — Sounds like a real person, not a textbook

## Spoken vs Written English

### ❌ Written/Academic (Avoid)

```
"I would like to emphasize the importance of..."
"Furthermore, it is crucial to note that..."
"Moreover, we should proceed with..."
"Therefore, in order to address this issue..."
"It is imperative that we establish..."
```

**Why avoid:**
- Too formal for speech
- Stiff and robotic
- Sounds like a consulting deck
- Unnatural in meetings/interviews

### ✅ Spoken/Natural (Prefer)

```
"The key thing is..."
"Let's focus on..."
"So let's move forward with..."
"Here's the issue..."
"We need to set up..."
```

**Why prefer:**
- Short and direct
- Natural rhythm
- Real meeting language
- People actually say this

## Red Flags

### Automatic 1–2 Score (Too Formal)

- "I would like to emphasize..."
- "It is crucial to note that..."
- "Furthermore..."
- "Moreover..."
- "Therefore, we should..."
- "In order to..." (when "to" works)
- "Utilize" (when "use" works)
- "Leverage" (when vague)
- Long passive sentences

### Borderline 3 Score (Stiff but Understandable)

- "Consequently, we..."
- "Additionally, we might..."
- "Regarding the issue of..."
- "With respect to..."
- "In the event that..."

### Good 4–5 Score (Natural)

- "So let's..."
- "Let's focus on..."
- "The thing is..."
- "Here's what I'm thinking..."
- "We should probably..."

## Phrase Length Guidelines

### Good Length (2–6 words)

- "push back on" ✅
- "loop in engineering" ✅
- "find middle ground" ✅
- "table this for now" ✅

### Too Long (7+ words)

- "I would like to push back on this" ❌
- "We should loop in the engineering team" ❌
- "Let's find a middle ground solution" ❌

**Fix:** Break into shorter phrases or simplify

## Professional vs Casual

### Professional (Appropriate for SpeakOps)

```
"I'd push back on that timeline."
"Let's loop in engineering."
"Can we table this for now?"
"I'm not sure that's the right approach."
```

### Too Casual (Avoid for SpeakOps)

```
"No way, that's crazy." ❌
"Yeah, whatever." ❌
"So like, we should..." ❌
```

### Too Formal (Avoid for SpeakOps)

```
"I respectfully disagree with said timeline." ❌
"We ought to involve engineering personnel." ❌
"It would be prudent to postpone this matter." ❌
```

## Industry Context

### IT/Product/AI Context (Good)

```
"under the hood" — Technical details
"at scale" — Production context
"latency vs accuracy" — Trade-off discussion
"baseline" — Reference point
"production-grade" — Quality level
```

### Generic Business (Less Relevant)

```
"circle back" — Generic (avoid unless specifically useful)
"touch base" — Generic (avoid unless specifically useful)
"move the needle" — Generic (avoid unless specifically useful)
```

## Cultural Considerations

### US/UK Tech Industry

- Direct but polite
- Confident but not aggressive
- Casual but professional
- Jargon is okay (tech context)
- Abbreviations are okay (tech context)

### What to Avoid

- Overly formal titles ("Mr./Ms." in meetings)
- Stiff politeness ("I would be grateful if...")
- Excessive deference
- Academic transitions
- Essay-style structure

## Common Mistakes

### Mistake 1: Over-politeness

❌ **Bad:** "I would be most grateful if you could possibly consider..."
✅ **Good:** "Could you take a look at this?"

### Mistake 2: Over-formality

❌ **Bad:** "Therefore, we ought to proceed with..."
✅ **Good:** "So let's go ahead and..."

### Mistake 3: Essay structure

❌ **Bad:** "First, X. Second, Y. Finally, Z."
✅ **Good:** "So first X, then Y, and finally Z."

### Mistake 4: Passive voice

❌ **Bad:** "It is recommended that..."
✅ **Good:** "I recommend..."

## Scoring Naturalness

When judging phrase naturalness (1–5):

**Score 1–2 (Reject):**
- Sounds written
- Sounds AI-generated
- Too formal for speech
- Robotic rhythm

**Score 3 (Borderline):**
- Understandable but stiff
- Can be improved
- Accept with revision

**Score 4–5 (Accept):**
- Natural spoken rhythm
- Appropriate formality
- Real meeting language
- People actually say this

## Examples

### Example 1: Stakeholder Pushback

❌ **Too formal:** "I must respectfully disagree with the proposed timeline."
✅ **Natural:** "I'd push back on that timeline."

### Example 2: Technical Discussion

❌ **Too formal:** "It is imperative that we establish a baseline."
✅ **Natural:** "We need to set a baseline."

### Example 3: Meeting Flow

❌ **Too formal:** "Furthermore, we should proceed to the next item."
✅ **Natural:** "So let's move on to the next item."

## Quick Reference

### Do ✅

- Use short phrases (2–6 words)
- Use contractions ("I'd", "let's", "can't")
- Use direct language
- Use natural transitions ("so", "anyway")
- Use confident but polite tone
- Use tech jargon appropriately

### Don't ❌

- Use "furthermore" or "moreover"
- Use "it is crucial to note"
- Use long passive sentences
- Use over-formal politeness
- Use essay transitions
- Sound like a consulting deck
